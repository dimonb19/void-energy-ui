# Phase Roadmap

This folder is the working plan for the UI system.

Rules for execution:
- One session = one phase.
- Finish the current phase before opening the next one.
- If a phase changes the contract, update the later phase docs before starting them.
- Do not treat every audit finding as mandatory. Follow the re-baselined contract from Phase 00.

Order:
1. `PHASE-00-contract-rebaseline.md`
2. `PHASE-01-boundary-contracts.md`
3. `PHASE-02-runtime-integrity.md`
4. `PHASE-03-docs-and-guardrails.md`
5. `PHASE-04-quality-hardening.md`

Intent:
- Phase 00 defines the real rules.
- Phases 01-04 implement against those rules.
